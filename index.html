This page tests the XHR pipeline.

The pipeline works like this:

When making ajax calls, calls are not fired until next tick (setTimeout 10ms) Therefore, any other calls being made meanwhile, will be stocked, and calls will be batched

Each call gets an ID that the endpoint is expected to return

The server must process calls in the received order. It's free to return some calls early should it want to. The client then opens up a new long-poll to the pipeline, where the server can put the next round of calls
